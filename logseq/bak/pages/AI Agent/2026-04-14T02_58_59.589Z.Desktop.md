	- ## Harness
		- References
			- [Ralph Wiggum as a "software engineer"](https://ghuntley.com/ralph/)
				- **Core loop**: `while :; do cat PROMPT.md | claude-code ; done` — one task per iteration, trust the LLM to pick what's most important
				- **Essential file stack** (loaded every loop):
					- `@PROMPT.md` — main instructions
					- `@fix_plan.md` — prioritized todo list, regenerate frequently
					- `@specs/*` — one spec per file, written before implementation
					- `@AGENT.md` — living doc of how to build/run/test; LLM self-updates it
				- **Two phases per loop**: generate code → backpressure (tests, linters, type checkers, compilers)
				- **Testing**: run tests immediately after each unit; include docstrings explaining *why* tests exist (future loops lose reasoning context)
				- **Static analysis is critical** for dynamic languages — use Pyrefly, Dialyzer, etc. or "you will run into a bonfire of outcomes"
				- **Prevent placeholders**: LLMs chase compilation rewards; explicitly instruct full implementations, search for TODOs and minimal stubs
				- **Search before creating**: always require "search codebase first, don't assume not implemented" — ripgrep can miss things
				- **Self-improvement**: let LLM update `@AGENT.md` with new learnings; document bugs in `@fix_plan.md` even if out of scope
				- **Loop back is everything**: design for LLM self-evaluation — add logging, compile and inspect output, feed results back into the next loop
				- **Subagents**: parallelize reads/searches (up to 500), but restrict builds/tests to single subagent to avoid backpressure collapse
				- **Context budget**: ~170k tokens per loop; primary context is the scheduler, subagents are the executors
				- **Git workflow**: commit after tests pass, push, tag on zero errors, increment patch version (start at 0.0.0)
				- **Recovery**: expect broken codebases; either `git reset --hard` or craft rescue prompts; try different models if stuck
				- **Specs drive correctness**: wrong code usually means wrong specs — check for contradictions
				- **Realistic expectations**: ~90% completion with Ralph, final 10% needs senior engineering judgment
			- [Anthropic: Effective harnesses for long-running agents](https://www.anthropic.com/engineering/effective-harnesses-for-long-running-agents)
				- an **initializer agent** that sets up the environment on the first run, and a **coding agent** that is tasked with making incremental progress in every session, while leaving clear artifacts for the next session.
				- You can find code examples in the accompanying [quickstart.](https://github.com/anthropics/claude-quickstarts/tree/main/autonomous-coding)
				- The Claude Agent SDK is a powerful, general-purpose agent harness adept at coding, as well as other tasks that require the model to use tools to gather context, plan, and execute. It has context management capabilities such as compaction, which enables an agent to work on a task without exhausting the context window.
